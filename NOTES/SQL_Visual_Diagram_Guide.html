#6 Combining Data â€” JOINs & SET Operators

Multiple tables ko milana â€” visual Venn diagrams ke saath

INNER JOIN
Sirf wo rows jo dono tables mein match karti hain. Unmatched rows dikhai nahi deti.
SELECT e.name, d.dept_name
FROM employees e
INNER JOIN departments d
ON e.dept_id = d.dept_id;
LEFT JOIN
LEFT table ki SARI rows + right table ki matching rows. Unmatched right = NULL.
SELECT e.name, d.dept_name
FROM employees e
LEFT JOIN departments d
ON e.dept_id = d.dept_id;
-- dept_name = NULL agar koi dept nahi
RIGHT JOIN
RIGHT table ki SARI rows + left table ki matching rows. Unmatched left = NULL.
SELECT e.name, d.dept_name
FROM employees e
RIGHT JOIN departments d
ON e.dept_id = d.dept_id;
-- emp.name = NULL agar koi emp nahi
SELF JOIN
Table apne aap se join hoti hai. Employee aur uske Manager ka naam ek hi table se nikalo!
SELECT e.name AS Emp,
m.name AS Manager
FROM employees e
LEFT JOIN employees m
ON e.manager_id = m.emp_id;
CROSS JOIN
Har row Ã— har row = Cartesian Product. 3 rows Ã— 4 rows = 12 rows result!
SELECT * FROM colors
CROSS JOIN sizes;
-- 3 colors Ã— 4 sizes = 12 combinations
FULL OUTER JOIN
Dono tables ki SARI rows. Koi match nahi toh NULL. MySQL mein direct support nahi!
-- MySQL workaround:
SELECT * FROM a LEFT JOIN b ON ...
UNION
SELECT * FROM a RIGHT JOIN b ON ...;

Q1
WHERE aur HAVING mein kya fark hai?
â­ Easy
WHERE individual rows ko filter karta hai â€” GROUP BY se pehle. HAVING groups ko filter karta hai â€” GROUP BY ke baad.
-- WHERE: Sirf IT employees ko consider karo
SELECT dept, COUNT(*) FROM employees
WHERE dept = 'IT'   -- Row filter (before group)
GROUP BY dept;

-- HAVING: 5 se zyada employees wale departments
SELECT dept, COUNT(*) FROM employees
GROUP BY dept
HAVING COUNT(*) > 5; -- Group filter (after group)
WHERE aggregate functions use nahi kar sakta (COUNT, SUM, AVG etc.) HAVING kar sakta hai
HAVING bina GROUP BY ke bhi likh sakte ho, lekin mostly GROUP BY ke saath use hota hai
Q2
2nd ya Nth highest salary kaise nikaalein?
â­â­ Medium
Do popular methods hain â€” interviewer ko dono batao!
-- Method 1: Subquery (Classic)

-- Method 2: DENSE_RANK (Modern, preferred)
SELECT salary FROM (
  SELECT salary,
    DENSE_RANK() OVER (ORDER BY salary DESC) AS rnk
  FROM employees
) t WHERE rnk = 2; -- N change karo Nth ke liye
DENSE_RANK method better hai kyunki N easily change kar sakte ho
RANK() use mat karo â€” gaps ki wajah se galat answer aa sakta hai
Q3
DROP, TRUNCATE, DELETE mein kya fark hai?
â­ Easy
Ye ek-ek kar ke samjhao â€” interviewer impress hoga!
-- DELETE: Row by row, rollback ho sakta, WHERE laga sakte

-- TRUNCATE: Ek baar mein sab, fast, rollback nahi

-- DROP: Table hi khatam, structure bhi jaata hai

DELETE = DML, row by row, ROLLBACK possible, WHERE support karta hai
TRUNCATE = DDL, ek saath sab, ROLLBACK nahi, identity reset hoti hai
DROP = DDL, table aur structure sab permanent delete!

UNION=duplicates remove (slow). UNION ALL=sab rakhta hai (fast). Dono mein same columns chahiye.
